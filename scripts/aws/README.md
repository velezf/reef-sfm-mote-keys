# `scripts/aws/` — AWS infrastructure layer

Bash scripts that build, operate, and tear down the EC2 GPU workstation for this project. Full reasoning lives in [`docs/aws-setup.md`](../../docs/aws-setup.md). This file is a one-page operator reference.

## First-time setup

Edit `config/aws-config.sh` — set `MY_IP_CIDR` to your current public IP in CIDR `/32` form. Look it up with `curl -s https://checkip.amazonaws.com`.

Then, from the repo root:

```bash
./scripts/aws/00-prereqs.sh           # verify aws CLI, credentials, DLAMI lookup
./scripts/aws/01-create-network.sh    # key pair, SG, EIP, secondary ENI
./scripts/aws/02-create-storage.sh    # 1 TB data EBS volume (standalone)
./scripts/aws/03-create-launch-template.sh   # launch template, AMI pinned
./scripts/aws/04-launch-instance.sh   # launch + attach EIP / ENI / data vol
./scripts/aws/05-first-boot-setup.sh  # mount data vol, set hostname (over SSH)
./scripts/aws/06-create-baseline-snapshot.sh   # snapshot pre-Chat-3
```

Total time: ~10–15 minutes; mostly waiting on instance-status checks.

After step 4, register the SSH alias:

```bash
IP=$(grep eip_public_ip docs/aws-resources.md | sed -E 's/.*`([^`]+)`.*/\1/')
sed "s/REPLACE_WITH_EIP/${IP}/" config/ssh-config.snippet >> ~/.ssh/config
```

You can now `ssh reef-ec2`.

## Day-to-day operations

```bash
./scripts/aws/start-instance.sh    # beginning of a work session
# ... do work ...
./scripts/aws/stop-instance.sh     # end of a work session
```

`start-instance.sh` verifies the license-critical state (secondary ENI attached, MAC unchanged, data volume attached) before handing back control. Read `docs/aws-setup.md` § "License stability" for why this matters.

## End of project

After Chat 9 ships v1.0:

```bash
./scripts/aws/teardown.sh   # interactive; preserves snapshots and (by default) data volume
```

## Idempotency

Every script in this directory is safe to re-run. The numbered scripts detect existing resources by `Name` and `Project` tags and reconcile rather than duplicate. Operational scripts (start, stop) detect current state and do nothing if already in the target state.

## Where resource IDs live

`docs/aws-resources.md` is generated by the scripts and contains all created AWS resource IDs (instance, EIP, ENI, volumes, snapshots, etc.). The file is git-ignored — re-run the appropriate script if a value is missing or wrong. The scripts use `read_resource KEY` from `lib.sh` to look up IDs from this file.

## File map

| File | What it does |
|---|---|
| `lib.sh` | Shared helpers: logging, tag args, resource persistence, state waits |
| `00-prereqs.sh` | Read-only diagnostic; no AWS resources created |
| `01-create-network.sh` | Key pair, security group, Elastic IP, secondary ENI |
| `02-create-storage.sh` | Standalone EBS data volume |
| `03-create-launch-template.sh` | Launch template (AMI pinned at build time) |
| `04-launch-instance.sh` | Run-instances, associate EIP, attach ENI, attach data vol |
| `05-first-boot-setup.sh` | Runs over SSH on the instance: format/mount data vol, hostname |
| `06-create-baseline-snapshot.sh` | Pre-Chat-3 multi-volume snapshot |
| `start-instance.sh` | Resume from stopped; verify license-critical state |
| `stop-instance.sh` | Stop instance to pause compute billing |
| `teardown.sh` | End-of-project cleanup; interactive, snapshot-preserving |
